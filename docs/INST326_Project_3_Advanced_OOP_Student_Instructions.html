Project Overview

Extend your existing system (Projects 1 & 2) by implementing inheritance hierarchies, polymorphic behavior, and composition relationships. This project demonstrates how advanced OOP principles enable code reuse, flexibility, and scalable architecture.

Key Goal: Transform your collection of independent classes into a sophisticated object-oriented system with meaningful inheritance relationships and polymorphic behavior.

Technical Requirements

1. Inheritance Hierarchy (Required)

Create at least one inheritance hierarchy with: - Base class (may be abstract) defining common interface - 2-3 derived classes that specialize the base class - Method overriding with appropriate use of super() - Logical relationships that reflect your domain

2. Abstract Base Classes (Required)

Implement at least one abstract base class using Python's abc module:

from abc import ABC, abstractmethod

class AbstractItem(ABC):
    """Base class for all library items."""

    @abstractmethod
    def calculate_loan_period(self):
        """Must be implemented by subclasses."""
        pass

3. Polymorphism (Required)

Demonstrate polymorphic behavior where: - Same method call produces different behavior based on object type - Code works with base class references but calls derived class methods - System handles multiple object types uniformly

4. Composition (Required)

Show at least one composition relationship: - One class contains instances of other classes - Demonstrate "has-a" relationships - Explain why composition was chosen over inheritance

Common Pitfalls to Avoid

‚ùå Inheritance for the sake of it - Only use inheritance when there's a true "is-a" relationship
‚ùå Deep hierarchies - Keep inheritance 2-3 levels maximum
‚ùå Missing super() calls - Always call parent constructors when appropriate
‚ùå Composition as afterthought - Plan which relationships should be composition vs. inheritance
‚ùå No polymorphism - Ensure derived classes actually override methods meaningfully
